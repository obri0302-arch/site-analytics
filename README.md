# Site Analytics — self-hosted аналог Hotjar

Збирає та візуалізує поведінку відвідувачів вашого сайту: heatmap кліків і руху миші,
глибину скролу, час на сторінці, rage clicks (швидкі повторні кліки = ознака зламаного UX),
час до першої взаємодії, точки виходу.

## Запуск сервера

```bash
cd site-analytics
npm install
npm start
```

Сервер піднімається на `http://localhost:3000`.
Дашборд: `http://localhost:3000/`
Трекер: `http://localhost:3000/static/tracker.js`

## Встановлення на аналізований сайт

Додайте перед `</body>`:

```html
<script
  src="http://localhost:3000/static/tracker.js"
  data-endpoint="http://localhost:3000"
  data-site="my-site">
</script>
```

- `data-endpoint` — адреса вашого сервера аналітики
- `data-site` — ідентифікатор сайту (якщо аналізуєте кілька сайтів одночасно)

Для продакшну: задеплойте сервер (наприклад на Render / Railway / VPS),
замініть `localhost:3000` на реальний домен. Дані пишуться у `data/events.jsonl`.

## Що показує дашборд

| Блок | Що дає для аналізу |
|---|---|
| **Heatmap кліків** | Куди реально тиснуть користувачі — порівняйте з тим, де ви хочете щоб тиснули (CTA, меню) |
| **Heatmap руху миші** | Проксі для уваги — куди дивляться, навіть якщо не клікають |
| **Rage clicks (жовті кільця)** | Місця, де щось виглядає клікабельним, але не працює — головний сигнал "слабкого місця" |
| **Глибина скролу** | Скільки % відвідувачів доходять до 25/50/75/90/100% сторінки — показує, де люди "відвалюються" і чи бачать вони контент нижче |
| **Топ елементів за кліками** | Що людям цікаво найбільше; що ігнорують (важливі кнопки з низьким CTR) |
| **Таблиця сторінок** | Перегляди, унікальні сесії, середній час, середня глибина скролу — порівняння сторінок між собою |

## Як інтерпретувати (типові висновки)

- **Високий rage-click показник на елементі** → елемент виглядає інтерактивним, але не є таким
  (неправильний курсор, нерозгорнутий акордеон, "мертва" кнопка) — фіксити в першу черговгу.
- **Глибина скролу падає різко на 25–50%** → перший екран не утримує увагу, або контент
  після нього виглядає "кінцем сторінки" (немає візуального натяку, що є ще контент).
- **Heatmap руху без кліків у зоні CTA** → люди бачать кнопку, але не натискають —
  проблема у формулюванні/контрасті/довірі (соц. доказ, ціна, незрозуміла вигода).
- **Кліки скупчені поза інтерактивними елементами** (текст, зображення, заголовки) →
  користувачі очікують, що це клікабельно (хочуть розгорнути/перейти) — варто зробити це посиланням.
- **Малий "avg time on page" + малий "avg scroll depth"** → або нерелевантний трафік,
  або перший екран не пояснює, що тут і чому варто залишитись.
- **Великий час до першої взаємодії** → сторінка повільно завантажується або
  головний CTA не помітний відразу.

## Що можна додати далі (roadmap)

- Реальні скріншоти сторінок під heatmap (через Puppeteer screenshot + canvas overlay)
- Сегментація по UTM/джерелу трафіку
- Запис session replay (rrweb)
- A/B порівняння двох версій сторінки
- Експорт у CSV/PDF звіт для клієнтів
